# API Contract

Base URL: `http://127.0.0.1:5000`

## `GET /health`

Назначение: проверить, что сервис запущен и обе обязательные модели загружены.

Response `200 OK`:

```json
{
  "status": "ok",
  "available_models": ["v1", "v2"],
  "models": {
    "v1": {
      "algorithm": "LogisticRegression",
      "threshold": 0.56
    },
    "v2": {
      "algorithm": "RandomForestClassifier",
      "threshold": 0.54
    }
  }
}
```

## `POST /predict`

### Headers

- `Content-Type: application/json`
- `X-Experiment-Key: <stable-key>` — optional, deterministic 50/50 routing when `model_version` is absent
- `X-Request-ID: <uuid>` — optional, otherwise generated by service

### Request Body

```json
{
  "model_version": "v2",
  "features": {
    "LIMIT_BAL": 20000,
    "SEX": 2,
    "EDUCATION": 2,
    "MARRIAGE": 1,
    "AGE": 24,
    "PAY_0": 2,
    "PAY_2": 2,
    "PAY_3": -1,
    "PAY_4": -1,
    "PAY_5": -2,
    "PAY_6": -2,
    "BILL_AMT1": 3913,
    "BILL_AMT2": 3102,
    "BILL_AMT3": 689,
    "BILL_AMT4": 0,
    "BILL_AMT5": 0,
    "BILL_AMT6": 0,
    "PAY_AMT1": 0,
    "PAY_AMT2": 689,
    "PAY_AMT3": 0,
    "PAY_AMT4": 0,
    "PAY_AMT5": 0,
    "PAY_AMT6": 0
  }
}
```

### Model Selection Rules

1. Если задан `model_version`, используется он.
2. Если `model_version` не задан, но есть `X-Experiment-Key`, версия выбирается детерминированно через `SHA-256(key) % 2`.
3. Если не задано ни то, ни другое, используется `v1`.

### Success Response `200 OK`

```json
{
  "prediction": 1,
  "probability": 0.786525,
  "model_version": "v2",
  "experiment_group": "test",
  "request_id": "2f933a17-c0e7-4968-b8b5-1da22468db8e"
}
```

Field semantics:

- `prediction`: бинарный класс `0` или `1`
- `probability`: вероятность класса дефолта
- `model_version`: фактически использованная версия модели
- `experiment_group`: `control`, `test` или `default_control`
- `request_id`: request tracing identifier

### Error Responses

Validation error `400`:

```json
{
  "error_code": "invalid_request",
  "message": "features.PAY_AMT6: Field required",
  "request_id": "fdf545b4-c03e-45f9-b97b-25032034d57d"
}
```

Invalid JSON `400`:

```json
{
  "error_code": "invalid_json",
  "message": "Request body must contain valid JSON.",
  "request_id": "fdf545b4-c03e-45f9-b97b-25032034d57d"
}
```

Invalid content type `400`:

```json
{
  "error_code": "invalid_content_type",
  "message": "Content-Type must be application/json.",
  "request_id": "fdf545b4-c03e-45f9-b97b-25032034d57d"
}
```

Internal error `500`:

```json
{
  "error_code": "internal_error",
  "message": "Internal server error.",
  "request_id": "fdf545b4-c03e-45f9-b97b-25032034d57d"
}
```

## Validation Rules

- Все 23 feature fields обязательны.
- Все feature fields должны быть целыми числами.
- Extra fields запрещены.
- Порядок признаков в runtime берётся из schema, а не из порядка ключей в JSON.
